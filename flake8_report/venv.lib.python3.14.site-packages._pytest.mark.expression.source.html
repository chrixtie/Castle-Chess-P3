1 r"""Evaluate match expressions, as used by `-k` and `-m`.
2  
3 The grammar is:
4  
5 expression: expr? EOF
6 expr: and_expr ('or' and_expr)*
7 and_expr: not_expr ('and' not_expr)*
8 not_expr: 'not' not_expr | '(' expr ')' | ident kwargs?
9  
10 ident: (\w|:|\+|-|\.|\[|\]|\\|/)+
11 kwargs: ('(' name '=' value ( ', ' name '=' value )* ')')
12 name: a valid ident, but not a reserved keyword
13 value: (unescaped) string literal | (-)?[0-9]+ | 'False' | 'True' | 'None'
14  
15 The semantics are:
16  
17 - Empty expression evaluates to False.
18 - ident evaluates to True or False according to a provided matcher function.
  • E501 Line too long (115 > 79 characters)

20 - or/and/not evaluate according to the usual boolean semantics.

255 class ExpressionMatcher(Protocol):

  • E501 Line too long (80 > 79 characters)
